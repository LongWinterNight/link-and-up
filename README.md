# Link-and-Up

Дашборд аналитики и планирования контента LinkedIn с прозрачным прогнозом вовлечения.
Превращает корпус постов (демо-набор из коробки или импортированный) в решения: что
реально собирает отклик, какие паттерны переиспользовать, что публиковать следующим — и
прогноз, который честно показывает свою неопределённость, а не прячет её.

Между «сырой аналитикой» и «постить по наитию» есть разрыв: цифры есть, а решения под
конкретного автора нет. Link-and-Up закрывает этот разрыв — считает эффективность приёмов,
ведёт банк идей и календарь, оценивает следующий пост по постам-аналогам и учится на
фактических метриках. Клиент работает целиком в браузере; импортированные данные хранятся
в `localStorage` и не покидают устройство.

## Возможности

- **Аналитика** — эффективность хуков, структур, CTA и эмоций; покрытие данных; корреляция реакций и комментариев
- **База знаний** — тематические кластеры, формулы победителей, анти-паттерны
- **Планирование** — банк идей, доска kanban и календарь каденса публикаций
- **Прогноз** — прозрачная эвристика с пошаговым разложением, бэктест leave-one-out собственной точности и честные состояния «недостаточно данных»
- **Петля обучения** — ввод реальных метрик своих постов калибрует модель
- **Гардрейлы** — настраиваемые правила brand-safety: `hard` блокирует публикацию и экспорт, `soft` предупреждает; свои правила (запрещённые термины, имена клиентов под NDA)
- **Мост идея → черновик** — каркас поста по формуле-эталону с проверкой гардрейлов
- **Импорт / экспорт** — двухшаговый импорт с дедупликацией; экспорт в JSON, CSV и Obsidian с редакцией по гардрейлам

## Принципы честности данных

- `0` означает *неизвестно*, а не ноль: посты без метрик не усредняются и не рисуются нулями
- Engagement rate = `комментарии / подписчики`, считается только там, где известно число подписчиков и комментарии ненулевые
- Прогноз возвращает диапазон, а не точку, и переходит в явное low-data состояние, когда основывать оценку не на чём
- Бэктест — leave-one-out по корпусу; при менее чем восьми измеренных постах он сообщает о собственной недостоверности, а не выдумывает число

## Технологический стек

- **Frontend**: React 18.3, TypeScript, функциональные компоненты
- **Сборка**: Vite 5
- **Состояние**: Zustand с `persist` (localStorage)
- **Графики**: SVG вручную, без графических библиотек
- **Виртуализация**: `@tanstack/react-virtual` для списков и таблиц
- **Темизация**: CSS custom properties, атрибут `data-theme`; светлая и тёмная — равноправны
- **Логика**: чистые функции в `src/lib/`, golden-set на Vitest (55 тестов)

## Быстрый старт

```bash
npm install
npm run dev        # dev-сервер
npm run build      # type-check + продакшн-сборка
npm run typecheck  # tsc --noEmit
npm test           # запуск тестов
```

## Структура проекта

```
src/
  lib/         чистая бизнес-логика (forecast, dedup, enrich, guardrails, exports, derive)
  tabs/        экраны верхнего уровня (Обзор, Аналитика, Посты, Кластеры, Идеи, Прогноз)
  components/  примитив Modal, PostModal, ImportModal, SettingsModal, charts, ui
  data/        демо-корпус
  store.ts     стор Zustand (состояние, действия, персистентность)
  types.ts     общие типы
```

## Тесты

Покрыты чистая логика (прогноз, калибровка, бэктест, дедуп, обогащение, гардрейлы,
экранирование CSV), интеграция стора (онбординг, пресеты, импорт, петля обучения) и
примитив модалки (focus-trap, Escape, возврат фокуса). Запуск — `npm test`.

## Статус

Pre-release. Дашборд работает на демо-корпусе и на импортированных данных. Бэкенд,
аккаунты и командные функции запланированы, но в этот клиент не входят.
